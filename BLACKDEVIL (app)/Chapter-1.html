===> iii. Flow Chart :

A graphical representation of an algorithm flowcharts is a diagram made up of boxes, diamonds and other shapes, connected by arrows.
    Each shape----> Step in process.
    Arrows----> the order in which they occur.

Rules/Guidelines.

    1. Draw a flow chart necessary requriments should to be listed out in logical order.
    2. Flow Chart should be clear, neat and easy to fllow.
    3. The direction of flow is left to right (or) top to bottom.

    4. Only one flow line sholud be entered in decision symbol, maximium three flow line should be leave.

    5. Only one flow line is used in conjuction with terminal symbol.

    6. Flow chart become complex better to use connected symbol.

    7. Ensure that flow chart has logical start and stop.
Example :
1. Leap year or not.
Algorithm :
    Step 1 : Start
    Step 2 : Read the year
    Step 3 : If (year mod 4)=0 then print "it is a leap year"
    Step 4 : Else print "It is not leap year".
    Step 5 : Stop.
Pseudocode :
    Start
    Read the year
    If (year % 4=0) Then write the year is "leap year"
    Else write the year is "not a leap year"
    Endif
    Stop
